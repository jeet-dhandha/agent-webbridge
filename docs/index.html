Kimi WebBridge with multiple Chrome profiles & logins at once

kimi-webbridge-fleet — drive several Google accounts through Kimi WebBridge simultaneously, not one at a time.

GitHub repo → npm package →

Kimi WebBridge (Moonshot AI) lets an AI agent drive your real Chrome with your real logins — but it has a single connection slot: one daemon, one extension, one profile. If you have a work account and a personal account, only one can be driven at a time. kimi-webbridge-fleet removes that limit without patching anything — one stock daemon per profile, a small router on the usual :10086, and your existing calls keep working. You just add a "profile" field.

Install

# As a CLI (npm)
npx kimi-webbridge-fleet profiles
npm i -g kimi-webbridge-fleet

# As a Claude Code skill/plugin
/plugin marketplace add jeet-dhandha/kimi-webbridge-fleet
/plugin install kimi-webbridge-fleet@kwb

Drive multiple profiles — same call, one extra field

kwb connect "Work" "Personal"   # point each extension at its own daemon (zero clicks)
kwb up "Work" "Personal"        # daemons + router on :10086, all connected

curl -s -X POST http://127.0.0.1:10086/command \
  -H 'Content-Type: application/json' \
  -d '{"action":"navigate","args":{"url":"https://mail.google.com"},"profile":"Work"}'

FAQ

Can Kimi WebBridge use multiple Chrome profiles at once?

Not on its own — stock Kimi WebBridge has a single connection slot, so exactly one Chrome profile is driven at a time. kimi-webbridge-fleet adds it: one stock daemon per profile plus a router on :10086, so every profile stays connected simultaneously.

Can Kimi WebBridge drive multiple Google accounts / logins at the same time?

Yes, with fleet. Each Chrome profile carries its own Google login, so one daemon per profile lets Gmail, Drive, Ads, and Search Console accounts all run side by side from one router on :10086.

Does Kimi WebBridge support multiple accounts simultaneously?

Stock Kimi WebBridge is built around a single primary browser and login. Multiple simultaneous accounts is exactly the gap kimi-webbridge-fleet fills — without patching the daemon or the extension.

Why does Kimi WebBridge only allow one login at a time?

The stock daemon enforces two singletons: it refuses to start if anything already answers :10086, and one daemon accepts exactly one extension connection. Fleet sidesteps both by running each profile's daemon on its own port behind a router.

How do I switch profiles in Kimi WebBridge without toggling the extension off?

Stock WebBridge makes you open chrome://extensions in the profile holding the slot and toggle WebBridge off — a few clicks every switch. Fleet points each profile's extension at its own daemon on disk with zero clicks and no popup, keeping all profiles live at once.

Is kimi-webbridge-fleet affiliated with Kimi / Moonshot AI?

No. It is an independent, MIT-licensed orchestration layer that supervises the stock kimi-webbridge daemon and reads Chrome's own files. It contains no Kimi WebBridge code.
